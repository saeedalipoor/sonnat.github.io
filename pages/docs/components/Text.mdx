<h2>
  <span>Separate the Semantics from the Style</span>
  <AnchorButton anchorId="separate-the-semantics-from-the-style" />
</h2>

<p>
  It’s important to realize that the style of a typography is independent from
  the semantic underlying element. Don't skip heading levels. In order to solve
  this problem, you need to separate the semantics from the style.
  <br /> You can change the text element's semantic with the <inlineCode>
    rootNode
  </inlineCode> property:
</p>

<DemoBox
  code={`{/* There is already an h1 in the page, let's not duplicate it. */}
<Text variant="h1" rootNode="strong">
  Heading Display 1
</Text>`}
>
  <Text variant="h1" rootNode="strong">
    Heading Display 1
  </Text>
</DemoBox>
